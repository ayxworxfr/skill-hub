# Debugging

## Core Principle

禁止症状层补丁。每次修复必须从现象追到可验证根因，再用最小正确改动修掉根因。**没有现象证据先收集；没有根因证据先验证；不许用兜底/忽略/跳过假装修好。**

只分析不改代码时，执行到根因和验证建议为止。

## 适用范围

优先用于：

- traceback、测试失败、运行时报错
- 结果不对、UI 行为异常、数据异常
- 本地正常但某环境异常
- 偶发失败、竞态、时序、缓存、配置差异
- 用户问"为什么会这样"/"定位一下"/"修这个问题"

不要用于：

- 没有现象但要重新设计：用 `planning`
- 行为正常但要整理结构：用 `refactoring`
- 添加新功能：用 `building`

## 强制流程

### 1. 确认现象

记录：

- 实际结果和预期结果
- 触发步骤、输入、环境、时间点
- 稳定复现还是偶发
- 最新错误、日志、截图、失败断言

没有现象证据时，先尝试复现或收集最接近证据，禁止凭描述猜测。

### 2. 建立证据链

读取并区分：

- **报错点**：traceback 里的最后一行 + 触发栈
- **调用方**：谁传入了导致错误的输入
- **数据来源**：异常值从哪里产生
- **配置/env/路径/依赖版本**：环境差异
- **最近改动或相关 diff**：bisect 候选
- **现有测试和边界用例**：哪些路径有保护

证据不足时写"不确定"，不要编造 API 或运行状态。

### 3. 中段自检

进入根因推导前确认：

- [ ] 报错点和调用方都有具体文件:行
- [ ] 异常输入的源头已经追到（不是停在"传进来就是错的"）
- [ ] 没把"看起来相关"的最近 diff 当结论，先验证再下定论

任一未通过 → 回 Step 2 补证据。

### 4. 追根因

输出内部判断必须区分：

```text
现象：用户看到什么
触发条件：什么输入/环境/状态导致
直接原因：哪段逻辑直接造成
根因：为什么会有这段错误逻辑、错误数据、错误边界或错误假设
验证根因：用什么证据证明它不是猜测
```

如果有多个假设，逐个验证或排除。**不允许多假设并存却不验证。**

根因必须能回答"这段代码为什么写成这样"或"这条数据为什么变成这样"，不能停在"这里 null 了"。

### 5. 修复根因

修复规则：

- 优先修错误假设、数据流、边界条件、状态机或配置映射
- 不用默认值吞掉异常
- 不跳过失败分支假装修好
- 不只捕获异常不处理原因
- 不保留失效旧逻辑
- 不顺手重构无关代码（如发现需要重构，停下切换 `refactoring`）

### 6. 验证修复

必须证明：

- 原复现路径不再失败
- 相关边界路径没有破坏
- 新增或现有测试覆盖本次问题
- lint/type/import/build 没有新增明显错误

详细验证策略见 `verifying` skill。**bug 修复必须有能复现原失败的测试或脚本，否则等同于未修复。**

### 7. Final Gate

输出前确认：

- [ ] 现象、触发条件、直接原因、根因、验证根因 五字段齐全
- [ ] 修复改的是根因层，不是症状层
- [ ] 复现路径已经验证不再失败
- [ ] 没有触发"禁止输出模式"任一行
- [ ] 残留写明（已知差异、未验证项），没有"应该没问题"

任一未通过不得声称修复完成。

## 检查门

以下情况必须停下补证据：

- 只知道报错行，不知道输入如何到达
- 只看到异常结果，没有解释为什么产生
- 修复依赖"兜底"/"忽略"/"跳过"
- 无法复现却直接改核心逻辑
- 验证没有覆盖原失败路径
- 多个假设并存却没有逐个验证

## 禁止输出模式

| 禁止输出 | 替代动作 |
|---|---|
| "可能是 X" | 给出验证 X 的具体步骤和证据 |
| "加个 try-catch" | 说明异常为什么发生，以及 catch 后如何恢复 |
| "先兜底一下" | 修复导致缺失/异常值的上游原因 |
| "测试通过应该好了" | 指出测试覆盖了原复现路径 |
| "环境问题" | 列出具体环境差异和验证方式 |
| "应该是 X 导致的" | 用证据验证 X，确认或排除 |
| "改了之后好像就好了" | 解释为什么这个改动修了根因 |

## 最终输出

修复类回答必须包含四项：

```markdown
## 根因

<一句话说明为什么会错，回答"这段代码为什么写成这样"或"这条数据为什么变成这样">

## 修复

<完整改动描述，不用"临时"/"先简单">

## 验证

- 复现命令/路径：
- 修复前结果：
- 修复后结果：
- 边界路径检查：

## 残留

<已知差异、未验证项、风险；没有写"无">
```
