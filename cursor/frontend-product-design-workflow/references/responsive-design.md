# Responsive Design

## 核心原则

响应式不是“加几个断点”，而是定义内容在不同容器宽度下如何重新排列、收缩、隐藏或滚动。

## 流程

1. 先找关键容器，不只看 viewport。
2. 定义最小可用宽度：通常从 320px 或项目目标设备开始。
3. 为每个布局区域定义窄屏行为：
   - 保留
   - 折叠
   - 换行
   - 隐藏
   - 滚动
4. 使用内容驱动断点：
   - 内容开始拥挤时断。
   - 操作按钮无法保持可点击时断。
   - 表格或代码块无法阅读时改为滚动。
5. 验证触摸目标、焦点可见和滚动区域。

## 常用策略

| 场景 | 策略 |
|---|---|
| 卡片矩阵 | `repeat(auto-fit, minmax(...))` |
| 表格 | 外层横向滚动，表头和关键列保持可读 |
| 动态列表项 | 保留内容槽位，缩小装饰槽或隐藏次要标识 |
| 工具栏按钮多 | 换行、折叠到菜单、保留主操作 |
| 双栏布局 | 窄屏变单栏，保持阅读顺序 |
| 长代码块 | 代码块内部滚动，不撑破页面 |

## 响应式契约模板

```text
容器：
最小宽度：
窄屏：
- 区域变化：
- 保留主任务：
- 溢出策略：
中屏：
- 区域变化：
宽屏：
- 区域变化：
不可破坏：
- 永远保留的槽位：
- 永远不允许的溢出：
```

## 验收宽度

默认至少说明：

- 320px：极窄移动端。
- 390px：常见手机。
- 768px：平板或窄桌面。
- 1024px+：桌面。

如果项目有明确目标设备，以项目目标为准。

## 阻断条件

- 固定宽度导致 320px 横向溢出但未说明。
- 触摸按钮小于 24px 且无等效点击区域。
- 视觉顺序和 DOM/focus 顺序明显冲突。
- 小屏隐藏主操作但没有替代入口。
