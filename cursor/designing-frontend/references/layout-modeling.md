# Layout Modeling

## 核心原则

布局不是样式微调，而是空间契约。先定义区域、槽位、尺寸和溢出规则，再写 CSS。

## 布局建模流程

1. 列出可见区域：header、sidebar、content、footer、toolbar、message row 等。
2. 给每个区域分配职责：导航、输入、内容、反馈、操作。
3. 判断布局维度：
   - 一维排列用 Flex：横向工具栏、头像 + 文本、按钮组。
   - 二维区域用 Grid：页面框架、三栏消息行、卡片矩阵。
   - 文档流优先：普通内容、文章、表单字段。
   - 定位只用于浮层、徽标、装饰、上下文菜单，不用于主布局。
4. 定义尺寸策略：
   - 固定槽：头像、图标按钮、侧边栏最小宽度。
   - 弹性槽：正文、消息内容、表格容器。
   - 内容槽必须显式处理 `min-width: 0`、换行、滚动或截断。
5. 定义溢出策略：
   - 文本：换行或截断。
   - 代码块：横向滚动。
   - 图片/视频：限制最大宽度和高度。
   - 表格：容器滚动，不撑破页面。
6. 定义响应式变化：
   - 哪些区域折叠？
   - 哪些区域换行？
   - 哪些区域隐藏但保留语义？
   - 哪些槽位永远不能被其他内容占用？

## Grid/Flex 判断门

| 场景 | 默认选择 | 原因 |
|---|---|---|
| 页面主框架、多列区域、明确槽位 | Grid | 同时控制行和列，区域边界清楚 |
| 头像 + 文本、按钮组、导航项 | Flex | 一维排列和对齐更直接 |
| 卡片自适应矩阵 | Grid | `auto-fit`、`minmax()` 更稳定 |
| 消息三槽、左右固定中间伸缩 | Grid | 头像槽与内容槽可物理隔离 |
| 浮层、tooltip、菜单 | Positioning | 脱离主文档流是需求本身 |

## 常见失败模式

| 失败模式 | 根因 | 修复动作 |
|---|---|---|
| 气泡跑进头像区 | 没有固定头像槽和内容槽边界 | 改成三槽 Grid 或明确 Flex 槽位 |
| 文本撑破容器 | 弹性子项缺少 `min-width: 0` 或溢出规则 | 给内容槽加收缩和换行/滚动规则 |
| 小屏横向溢出 | 固定宽度过多，未定义窄屏降级 | 改为流式宽度、折叠或滚动容器 |
| 视觉靠 margin 拼 | 主结构没有父级布局模型 | 回到容器定义 Grid/Flex |
| 改一个状态其他状态崩 | 没有状态矩阵 | 先补组件状态和内容边界 |

## 阻断条件

存在以下情况时，不允许继续写视觉样式：

- 主布局依赖 `position: absolute` 完成普通排列。
- 用负 margin 或魔法 padding 修复区域错位。
- 没有说明长文本、图片、代码块、空状态如何处理。
- 不知道每个区域的固定/弹性/溢出策略。
