# UI Design Process

## 核心原则

视觉设计必须服务任务、结构和状态。不要先选颜色、圆角、阴影；先定义信息层级和布局密度。

## 设计流程

1. 明确用户任务：
   - 用户来到这个界面要完成什么？
   - 主操作是什么？
   - 哪些信息只辅助决策？
2. 建立信息层级：
   - Primary：最重要内容或操作。
   - Secondary：辅助说明、元信息、次要操作。
   - Tertiary：背景、装饰、弱提示。
3. 定义视觉语气：
   - 专业工具：高可读、低装饰、强状态反馈。
   - 协作/实时界面：阅读舒适、节奏清楚、输入区域稳定。
   - 数据密集：密度可控、表格/筛选/状态优先。
   - 营销展示：品牌记忆点、视觉冲击、转化路径。
4. 选择密度：
   - 紧凑：信息密集、操作频繁、适合工具。
   - 标准：大多数产品页面。
   - 舒展：阅读、展示、低频操作。
5. 使用 token：
   - 颜色：background、surface、border、text、muted、accent、danger。
   - 间距：统一 spacing scale，不随手写随机 px。
   - 字体：正文、标题、说明、代码分别定义。
   - 圆角和阴影：表达层级，不当装饰堆叠。
6. 定义状态表现：
   - loading：骨架、spinner、流式占位。
   - empty：说明原因和下一步行动。
   - error：错误原因、恢复操作、重试入口。
   - disabled：为什么不可用，是否需要提示。
   - success：确认反馈，不打断主流程。

## 视觉检查表

- 主内容是否一眼能识别？
- 主操作是否比次操作更突出？
- 间距是否表达了分组关系？
- 颜色是否有语义，而不是随机装饰？
- 状态反馈是否清楚？
- 小屏幕上主任务是否仍然可完成？
- 动效是否帮助理解状态变化？

## 禁止模式

| 禁止表述 | 必须替换为 |
|---|---|
| 现代简洁 | 具体说明字体、间距、层级、色彩角色 |
| 更有高级感 | 说明用什么视觉手段降低噪音或提升重点 |
| 加点动画 | 说明动画触发条件、持续时间和用途 |
| UI 美化 | 说明要改善的是层级、密度、反馈还是品牌感 |

## 动态内容界面设计重点

- 阅读节奏优先于装饰。
- 不同来源或不同状态的内容要有稳定空间关系。
- 流式更新状态要清楚但不能抢焦点。
- 输入区或主操作区必须稳定，不因内容滚动而跳动。
- 代码块、引用、工具结果要有独立视觉层级。
- 错误消息要提供重试或恢复路径。
