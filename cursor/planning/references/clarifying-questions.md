# Clarifying Questions（决策驱动变量识别详解）

主文 Step 2.5 已规定 driver 三分类处理 + 先查再问顺序 + 硬门。本 reference 补充 AskUserQuestion 的具体用法、反例、子模式 driver 速查、输出片段。

## driver 判定标准

| 情况 | 判定 |
|---|---|
| 变量取不同值时，最终推荐**会变**（候选切换 / 切片顺序变 / 选型不同）| **driver**，必须解决 |
| 变量取不同值时，推荐**不变**或仅细节变 | **边角**，可标"假设" |

driver 典型（必须问或必须查）：

- 选型类：目标平台、用户量级、性能预算、团队规模 / 技术栈、是否跨平台
- 设计类：是否改公开接口、是否影响数据迁移、并发模型
- large 类：是否阻塞他人、灰度需求、上线时间窗、切片粒度偏好
- deps 类：许可证约束、已有依赖、维护成本承受度、安全合规要求

边角典型（可假设）：命名风格、注释语言、import 顺序、不影响推荐的细节配置（日志格式 / 编辑器配置）。

## AskUserQuestion 用法

参考 Anthropic AskUserQuestionTool / Spring AI 共识：

- **优先多选**：列 2-4 个候选答案，每个 description 写"选这个意味着什么"（implication）
- **每条 question 短**：question ≤25 字；header ≤12 字；options.label ≤5 字
- **一次最多 4 个问题**：超过拆两轮
- **问题独立**：不要问"如果你选 A 那么 B 选什么"（链式依赖问题分轮问）
- **推荐项放第一**：标 "(Recommended)"，给默认下沉路径

## 问什么 / 不问什么

问：

- 选型 driver（平台 / 规模 / 已有依赖 / 兼容约束）
- 大方向 trade-off（速度 vs 可维护 / 通用 vs 定制 / 自研 vs 三方）
- 不可推断的硬约束（死线 / 合规 / 关键决策偏好）

不问：

- 项目代码里能查到的（dependencies、版本、目录结构）
- 用户在原始 prompt 已说过的
- memory 中已有的稳定偏好
- 边角细节（命名风格 / 注释语言 / 不影响推荐的小事）
- 一次问 5 个以上（拆轮）

## 何时不该问（避免无限提问）

- 用户明确说"按你判断"/"看你"→ 推断 + 引用证据 + 列出风险，不再问
- 已问过 2 轮仍信息不全 → 输出"暂不推荐"，列还缺什么，停下
- driver 已被项目事实回答 → 直接引用，不重复问
- 生态只有单一事实标准 → 说明"无可比"，跳过 driver 提问

## 反例：微信小游戏方案卡

错误做法：

```text
假设：游戏类型为 2D 休闲 / 轻度游戏
假设：团队规模为 1-3 人，有前端 / TypeScript 经验
假设：目标平台为微信小游戏，不要求跨平台发布 App
```

三条都是 driver：

- 游戏类型决定 Cocos / Phaser / LayaAir 排序（休闲 vs 重度 vs 3D 推荐完全不同）
- 团队 TS 经验决定文档生态权重和上手成本
- 是否跨 App 决定是否保留 Cocos Creator 这类支持原生导出的引擎

正确做法（列候选前用 AskUserQuestion）：

- Q1：游戏类型主要是哪类？ 选项 = 2D 休闲 / 2D 中重度 / 3D / 多人实时；description 写每类对引擎权重的影响
- Q2：是否需要同时上 App（iOS/Android 原生）？ 选项 = 仅微信小游戏 / 同时上 App / 暂未定
- Q3：团队游戏引擎经验？ 选项 = 仅 TS Web / 有 Cocos / 有 Unity/UE

收到答案后再做候选对比。

## 各子模式 driver 速查

| 子模式 | driver 典型变量 |
|---|---|
| design | 影响范围 / 调用方约束 / 性能 vs 可维护偏好 / 是否改公开接口 |
| large | 灰度需求 / 是否阻塞他人 / 上线窗口 / 切片粒度偏好 |
| deps | 平台 / 规模 / 已有依赖 / 许可证约束 / 维护成本承受度 / 是否跨平台导出 |

## 输出片段

在"意图与边界"之后、"项目事实"之前追加：

```markdown
## 决策驱动变量

| 变量 | 类别 | 取值 | 来源 |
|---|---|---|---|
| 游戏类型 | driver | 2D 休闲 | 用户回答（AskUserQuestion Q1） |
| 跨平台 | driver | 仅微信 | 用户回答（AskUserQuestion Q2） |
| 团队经验 | driver | TS Web | memory: user_role.md |
| 命名风格 | 边角 | camelCase | 假设（与现有代码一致） |
```

边角不必逐条列；driver 必须全部列出并标来源。
